[[auth]]
== 登录与授权
当前采用OAuth2的机制处理用户请求，Token的实现方式采用JWT标准。当前所有医生端client在申请token时需要采用client_id为
`doctor_app` 的Basic Auth验证机制，Client credentials为空。即所有申请token的请求
都需要带一个 `Authorization` 的header，值为 `Basic ['doctor_web:'的base64编码]`。具体的请求种类见下文。

=== POST /api/v0/oauth/token
申请Access Token与Refresh Token。

医生客户端提供登录用户名和密码，调用此接口。成功后返回 `access_token` 和 `refresh_token`。
在用户登录成功后续的http请求中，需要带上 `access_token`。如果token过期，则服务器端会提示
客户端token已过期，如：

----

此时客户端带上之前保存的 `refresh_token` 请求更换新 `access_token`。
如果成功客户端则得到一个新的 `access_token`。如果由于 `refresh_token` 也失效或者其他原因
更新不成功，则客户端应该跳转到登录界面，提示用户重新输入用户名密码，客户端再次调用此接口申请
`access_token` 即可。

客户端在用户退出登录时清空本地保存的 `access_token` 和 `refresh_token` 并需要调取
`/oauth/revoke` 接口。

注意整个过程任何时刻客户端都不应保存任何形式的用户密码，而应该保存 `access_token` 和
`refresh_token`。目前默认设置 `access_token` 过期为12个小时，`refresh_token` 过期为
30天(开发环境分别为5分钟／1小时)。

以下部分分别就申请新token和更新token作详细说明。

==== Request Header
|===
|Key|Value|Note

|`Authorization`
|`Basic ZG9jdG9yX3dlYjo=`
|Basic 后面的值为 `doctor_web` 这个client_id与空密码 `''` 通过 `:` 拼接后得到 `doctor_web:`
再经过base64编码后的结果。客户端调用此接口必须具备此client_id用来区分患者端和医生端。

|===

==== Request Body
可以通过form-urlencoded 或 form data 的方式提供。
请求格式分为申请新的token和刷新过期token两种类型。

===== 申请Access Token

|===
|Parameter|Value|Description|Note

|`grant_type`
|必须为 `password`
|授权的类型
|required

|`username`
|用户手机号
|
|required

|`password`
|用户密码
|
|required

|`device_type`
|设备类型，`ios` 或 `android`
|
|optional

|`device_token`
|设备ID
|
|optional

|===

* Example Request

[source,bash]
----
$ curl -X POST -H "Authorization: Basic ZG9jdG9yX3dlYjo=" -d 'grant_type=password&username=18600000001&password=testpw' "http://172.31.2.37:9004/api/v0/oauth/token"
----

=== POST /api/v0/oauth/revoke

类型解除设备的绑定关联。

成功后返回success；如果该token已过期或不存在，也会返回成功success。
